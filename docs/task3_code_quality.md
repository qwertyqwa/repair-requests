# Задание 3 — качественные характеристики кода

## 1. Полнота обработки ошибочных данных

- Валидация обязательных полей заявки и телефона: `repair_requests/routes.py`
- Валидация количества комплектующей и подтверждение удаления: `repair_requests/routes.py`, `repair_requests/static/app.js`
- Сообщения пользователю (success/error/info/warning) через `flash()`: `repair_requests/routes.py`

## 2. Наличие тестов

- Юнит‑тест функции среднего времени ремонта: `tests/test_stats.py`
- Сквозные сценарии (роли, создание заявки, комментарии/комплектующие, уведомления): `tests/test_web_app.py`

## 3. Средства контроля корректности входных данных

- Серверная валидация форм: `repair_requests/routes.py`
- Ограничения в БД (UNIQUE/CHECK/FK): `repair_requests/schema.sql`

## 4. Средства восстановления при сбоях

- Резервное копирование БД: `scripts/backup_db.py`
- Восстановление: замена файла `data/app.db` на резервную копию.

## 5. Наличие комментариев и описаний основных функций

- Код стремится быть самодокументируемым (именование, небольшие функции).
- Основные алгоритмы и спецификация оформлены в `docs/assignment1.md` и `docs/task1.md`.

## 6. Проверка корректности передаваемых данных

- Проверка прав доступа по ролям и принадлежности заявки мастеру: `repair_requests/routes.py`, `repair_requests/store.py`
- Ссылочная целостность в SQLite: `repair_requests/schema.sql`, `repair_requests/db.py`
